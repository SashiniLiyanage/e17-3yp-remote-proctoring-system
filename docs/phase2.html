PHASE 2 :Develop the frontends

User Interfaces

Students' Desktop App

The student can create an account in the platform by registering to the service.

  • The registering process will go through a verification step where an email will be sent with a verification code to the user
  • Once the student verifies the account by entering the 6 digit code, he/she can log into the system where it will be directed to the home page.
  • Within the desktop application, the student can do the following tasks,

    • See the remaining time for the next exam
    • Test the camera, microphone and speaker before an exam
    • Connect to an examination room
    • In the examination room, student can see the proctor/(s), see warning messages and chat/talk with the proctor
    • See the examination schedule
    • See the courses the student is currently registered to

Note : Since the backend is not implemented yet, you can enter any verification code for demonstration purposes.

User Interfaces

Proctors' Browser App

The proctor can create an account in the platform by registering to the service.

  • The registering process will go through a verification step where an email will be sent with a verification code to the user
  • Once the proctor verifies the account, he/she can log into the system where it will be directed to the home page.
  • Within the browser application, the proctor can do the following tasks,

    • Schedule examinations with appropriate meeting link
    • Check the examination schedule and edit them if necessary
    • Connect to an examination room
    • In the examination room, the proctor can examine the students closely, detect unauthorized behaviors, send broadcast messages and talk to students directly
    • Check the summary of the recent examinations proctored in the dashboard
    • Keep track of the relevant courses and details about their examinations

Note : Since the backend is not implemented yet, you can enter any verification code for demonstration purposes.
